# Copy System — Framework Detalhado

Leia este arquivo ao executar o **Modo 0** do Escriba. Ele detalha como construir cada uma das 5 seções do documento Copy System.

---

## Regra de Ouro

O Copy System é escrito **na voz da marca**, não sobre ela. O documento já é um exemplo vivo do tom que define. Se a marca é direta e irreverente, o Copy System é direto e irreverente. Se é sofisticada e técnica, o documento reflete isso.

---

## Seção 1 — Tom de Voz

**O que entregar:**

1. **3–5 adjetivos que definem o tom** — seja específico; evite genéricos ("profissional", "amigável"). Prefira pares de tensão que capturam a nuance: "direto mas empático", "provocador mas respeitoso".

2. **O que a marca diz** — 3–5 frases de exemplo que capturam a voz. Escreva frases reais, não descrições.

3. **O que a marca nunca diz** — 3–5 anti-exemplos. Inclua o porquê de cada restrição (conflita com valor X, soa como concorrente Y, afasta avatar Z).

4. **Nível de formalidade** — escolha um ponto no espectro: Formal / Semi-formal / Coloquial / Íntimo. Justifique com base no ICM.

5. **Relação com o leitor** — como a marca se posiciona em relação ao cliente: Parceiro, Autoridade, Mentor, Provocador, Aliado, Guia. Explique a dinâmica.

**Formato de output:**
```
## Tom de Voz

**Adjetivos:** [adjetivo 1], [adjetivo 2], [adjetivo 3], [adjetivo 4]

**A marca fala assim:**
- "[frase exemplo 1]"
- "[frase exemplo 2]"
- "[frase exemplo 3]"

**A marca nunca diz:**
- "[anti-exemplo 1]" — porque [razão]
- "[anti-exemplo 2]" — porque [razão]

**Formalidade:** [nível] — [justificativa de 1 frase]

**Relação com o leitor:** [tipo] — [descrição de 1–2 frases de como isso se manifesta]
```

## Seção 2 — Mensagens Centrais por Avatar

Para **cada avatar identificado no ICM**, construa um bloco com 4 elementos:

1. **Dor principal em linguagem do cliente** — use as palavras exatas que o avatar usa. Não traduza para linguagem de produto. Se o ICM tem falas reais, use-as.

2. **Transformação prometida** — o "depois" em linguagem do cliente. Não é a feature, é o estado emocional e prático que a pessoa vai alcançar.

3. **Objeção mais comum + como rebater** — a resistência que aparece mais frequentemente para esse avatar. O rebuttal deve ser específico, não genérico ("custa muito" → "você gasta X hoje em Y, que não resolve; aqui você resolve por Z").

4. **Frase que resume o porquê de comprar** — uma frase que o próprio avatar usaria para explicar para um amigo por que decidiu comprar. Deve soar como ele, não como a marca.

**Formato de output:**
```
## Mensagens Centrais por Avatar

### Avatar: [Nome do Avatar]

**Dor principal (em linguagem do cliente):**
"[frase na voz do cliente]"

**Transformação prometida:**
De: [estado atual em linguagem emocional]
Para: [estado desejado em linguagem emocional]

**Objeção mais comum:** "[objeção exata]"
**Como rebater:** [rebuttal específico — 2–3 frases]

**Frase de compra (em voz do cliente):**
"[frase que o cliente usaria com um amigo]"
```

## Seção 3 — Ângulos por Estágio de Consciência

Use os **5 estágios de Eugene Schwartz**. Para cada estágio, entregue 3 elementos:

1. **Qual ângulo de copy funciona** — a abordagem estratégica (problema? solução? comparação? urgência?)
2. **Gancho de abertura** — uma frase de exemplo que captura a atenção desse estágio
3. **CTA adequado** — o próximo passo que faz sentido para quem está nesse estágio

**Os 5 estágios:**

| Estágio | Quem é | Foco do copy |
|---------|--------|--------------|
| 1. Sem Consciência | Não sabe que tem o problema | Despertar — mostre o problema antes de falar de solução |
| 2. Consciente do Problema | Sabe que tem o problema, mas não conhece soluções | Ampliar a dor, validar a frustração, abrir o horizonte de possibilidades |
| 3. Consciente da Solução | Conhece soluções, mas não conhece seu produto | Posicionar sua solução como a abordagem certa |
| 4. Consciente do Produto | Conhece seu produto, mas ainda não comprou | Remover objeções, criar urgência, destacar diferenciais |
| 5. Mais Consciente | Já quer comprar, quer a melhor oferta | Oferta direta, prova social, garantia |

**Formato de output:**
```
## Ângulos por Estágio de Consciência

### Estágio 1 — Sem Consciência
**Ângulo:** [estratégia]
**Gancho:** "[frase de exemplo]"
**CTA:** [próximo passo adequado]

### Estágio 2 — Consciente do Problema
[repetir estrutura]

[... para todos os 5 estágios]
```

## Seção 4 — Ganchos Validados

Entregue **10–15 ganchos de abertura** derivados diretamente das dores e desejos do ICM.

**Regras:**
- Cada gancho deve ter no máximo 2 frases
- Organize por avatar e estágio de consciência
- Inclua nota de sub-segmento: para qual perfil específico dentro do avatar esse gancho tende a ressoar melhor
- Varie os formatos: pergunta, declaração, dado surpreendente, história curta, provocação

**Formato de output:**
```
## Ganchos Validados

### Avatar: [Nome do Avatar]

**Estágio: [Estágio]**

1. "[gancho]"
   → Sub-segmento: [quem tende a responder melhor]

2. "[gancho]"
   → Sub-segmento: [quem tende a responder melhor]

[... repetir para outros avatares e estágios]
```

## Seção 5 — Restrições e Sensibilidades

Este é o filtro de qualidade de toda comunicação futura do projeto.

**O que entregar:**

1. **Palavras/expressões proibidas** — lista de termos que conflitam com o Brand Core ou que o ICM rejeita. Para cada um, explique o porquê (soa genérico, remete ao concorrente X, contradiz valor Y da marca).

2. **Promessas proibidas** — afirmações que a marca nunca pode fazer (por limitação legal, ética, ou porque contradiz o posicionamento).

3. **Abordagens proibidas** — estilos ou táticas de copy que são incompatíveis com o tom ou com o avatar (ex: urgência falsa, autoridade exagerada, linguagem de infoproduto barato).

4. **Sensibilidades do avatar** — tópicos que o ICP rejeita ou que precisam de cuidado especial (ex: comparações diretas com concorrentes, linguagem de fracasso pessoal, promessas de enriquecimento rápido).

**Formato de output:**
```
## Restrições e Sensibilidades

**Palavras/expressões proibidas:**
- "[termo]" — porque [razão]
- "[termo]" — porque [razão]

**Promessas proibidas:**
- [promessa] — porque [razão]

**Abordagens proibidas:**
- [abordagem] — porque [razão]

**Sensibilidades do avatar:**
- [tópico] — como tratar: [orientação]
```

## Checklist do Copy System antes de entregar

- [ ] O documento está escrito na voz da marca (não sobre ela)?
- [ ] Cada seção é acionável — o operador consegue abrir durante a produção e encontrar a resposta?
- [ ] Os ganchos estão organizados por avatar E estágio?
- [ ] As restrições têm justificativa (não são apenas listas de veto)?
- [ ] A Seção 2 usa linguagem do cliente, não do produto?
- [ ] Os 5 estágios de Schwartz estão cobertos na Seção 3?
